1. git config
Использование: git config –global user.name “[name]”
или же:        git config –global user.email “[email address]”
Эта команда указывает имя автора или его электронный адрес для последующего использования коммитов.

2. git init
Использование: git init [repository name]
Эта команда используется для использования новой директории.

3. git add
Использование: git add [file]
Команда git add добавляет содержимое рабочего файла в индекс.

4. git status
Использование: git status
Данная команда показывает файлы и их рабочее состояние.

5. git commit
Использование: git commit -m “[ Type in the commit message]”
Команда git commit берёт все данные, добавленные в индекс с помощью git add, и сохраняет их во внутренней базе данных.

6. git reset
Использование: git reset [file]
Команда git reset используется в основном для отмены изменений, а также изменяет состояние индекса.

7. git diff
Использование: git diff
Команда git diff используется для вычисления разницы между любыми двумя Git ветками.

8. git push
Использование: git push [variable name] master (master может быть заменён на main)
Эта команда отправляет совершённые изменения на удалённую репозиторию. (в нашем случае github)

9. git rm
Использование: git rm [file]
Эта команда удаляет файл с рабочей директории и показывает этапы удаления.

10. git init
Использование: git init [repository name]
Эта команда используется для использования новой директории.
